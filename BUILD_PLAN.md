# BUILD_PLAN - agent brief

This file is both documentation and a brief for an agent (an automation agent / Cowork) to
**publish**, **extend**, and **add to** this repo. Read it, then follow the conventions
exactly.

## Purpose

`omics-demos` is a set of public, clean-room toy examples derived from private research
repos. Each demo teaches one idea with synthetic data. The private repos are **never**
read from or copied - every demo is a fresh, generic reimplementation.

## Hard rules

1. **Synthetic data only.** No real sequencing data, no proprietary code, no internal
   identifiers, sample names, or paths. Generate everything in-repo.
2. **One command to run.** Each demo runs from repo root via a `make <target>` and exits 0.
3. **Self-contained.** No downloads at run time. `pip install -r requirements.txt` is the
   only setup. HTML demos open with no server (data inline).
4. **Every demo has:** a `README.md` (what it shows, `## Run`, `## What it does`, real
   `## Example output`, a committed preview under `assets/`) and a short, readable
   implementation. Keep dependencies to numpy/pandas/matplotlib unless a demo genuinely
   needs more (e.g. `pylabrobot` for `liquid-handling`, `scipy` for `rna-seq`).
7. **Shared style.** All plots import `omics_style.py` (repo root) and call `S.apply()` for the
   house look: Manrope (bundled in `.fonts/`), a baby-pastel palette with fine-line outlines,
   and constrained layout so text never overlaps. Keep new demos on it.
5. **Committed artifacts:** plots under `*/assets/*.png` are committed so READMEs render.
   Generated data under `*/data/` is git-ignored and regenerated by `make`.

## Publishing (do this in an automation agent, with your own gh auth)

Create the repo under the `di-omics` account and push. Nothing here should be pushed by
an assistant without your review - you run these:

```bash
# from the repo root (the omics-demos folder)
git init -b main
git add .
git commit -m "omics-demos: clean-room toy examples (synthetic data)"

# option A - GitHub UI: create empty repo di-omics/omics-demos (no README), then:
git remote add origin git@github.com:di-omics/omics-demos.git
git push -u origin main

# option B - gh CLI, one shot:
gh repo create di-omics/omics-demos --public --source=. --push
```

Optional: enable **GitHub Pages** (Settings -> Pages -> deploy from `main`) so
`chromatin-browser/index.html` gets a live URL.

## Adding a new demo (template)

1. `mkdir -p <demo>/assets`
2. Write `generate_data.py` - synthesize inputs (seed the RNG for reproducibility).
3. Write `analyze.py` - compute the metric/result; print a compact summary; write a small
   output table to `data/`.
4. Write `plots.py` - one clear figure to `assets/<demo>_qc.png`.
5. Add a `make <target>` in the top-level `Makefile`.
6. Write `<demo>/README.md` following the template in the other demos.
7. Verify: from repo root, `make <target>` runs clean and regenerates the preview.
8. Add a section + preview to the top-level `README.md`.
9. Commit.

### `rna-seq` (built - kept for reference)
Bulk/single-blastocyst RNA-seq mini-pipeline on synthetic counts:
- `generate_data.py`: gene × sample count matrix for two conditions with a planted DE set.
- `analyze.py`: CPM/log-normalize, PCA, a simple per-gene t-test with BH correction; report
  how many planted DE genes are recovered.
- `plots.py`: PCA scatter + volcano.
Keep it distinct from `flex-rna` (probe-based).
